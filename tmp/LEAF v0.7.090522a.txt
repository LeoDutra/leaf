LEAF v0.7.090522a

NEWS:

	DOMElement:

	- getters now return null if no element in DOMElement instance;
	- renamed addElement and addChild to appendElement and appendChild;
	- renamed invertAxys to invertXY;
	- added: dispatchEvent(type), getChildElements(), getFirst(), getLast(),
  	  getPrevious(), getNext(), cloneChildElements(), removeChildren(), purgeChildren(), setAttribute(attribute, value),
  	  getAttribute(attribute) and appendChildren(childNodes);

	General:
	- removed DOM.core.isIE (now using real time quick detection);
	- fixed DOM.purgeElement (error would occur if no element);
	- removed DOM.importXML. A new version will be built (it was evil with loading pages);
	- removed Object.inherit;
	- added Object.extend(object, sourceObject, doOverride=true);
	- added Array prototype, which has an iterator (each(array, itemHandler)) for general list interactions;
	- speedup in DOM.core.getElement;
	- speedup in DOM.getByTag;
	- speedup in DOM.getByClass;
	- speedup in DOM.core.hasCollision;
	- speedup in Mouse.getPosition;
	- speedup and fixes in DOMElement getOpacity and setOpacity;
	  (FIXME: problem for IE static elements with no dimension set);
	- DOM.getById returns only found elements now and null if there is no matches;
	- more ActiveX versions to AJAX.createRequester;
	- removed setParent, setChild so use external setElement(getChild/getParent) to set;



NEEDS:

	- see getOpacity before commit
	- fix semantics (o, e, k, etc)
	- general test
	- try optimization using childNodes on DOM.getByTag
	- research for more versions of aX to DOM.buildxml
	- DOMElement.getOffset, cannot reference offsetParent on IE6 elements before append (an error strikes)
	- JSON handler
	- research currentStyle (IE/OP) and window.getComputedStyle(e, null)(FF/OP) use when getting style property null for common style (<element>.style)



OBS:

	- instanceof is faster than o.constructor ===
	- function.apply(this, arguments) is slower than function.call, that is slower than set each attribute with this
	  (apply > call > object.property)
	- use 15ms as minimum delay for setTimeout e setInterval when animating
